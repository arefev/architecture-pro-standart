# FURPS

| Код | Требования                         | Комментарий  |
|-----|------------------------------------|--------------|
| F   | Функциональные (Functionality)     |              |
| F1    | На сайте должнен выводиться список доступных депозитов с актуальными ставками                               |              |
| F2    | На сайте пользователь может подать заявку на депозит, оставив свой номер телефона и ФИО. При отправке формы, данные на сервер должны попадать в зашифровонном виде                                |              |
| F3    | В интернет банке для авторизованного пользователя должен выводиться список доступных депозитов с актуальными ставками и персонализированные ставки                                |              |
| F4    | В интернет банке авторизованный пользователь может подать заявку на открытие депозита, указав счет и сумму депозита. Операция должна подтверждаться с помощью СМС кода. Так же отправляемые данные должны шифроваться                                |              |
| F5    | В АБС необходимо реализовать хранение и отображение ставок по депозитам. С функционалом могут работать сотрудники бэк офиса депозитов и кредитов.                                  |              |
| F6    | На сайте пользователь видит предложение открыть кредит по некоторой ставке                                 |              |
| F7    | Клиент может подать заявку, оставив свой номер телефона и Ф. И. О. Если клиент ещё заполнит паспортные данные, то предполагается сразу выдать ему решение об одобрении или неодобрении ему кредита                                |              |
| F8    | Менеджер отделения банка может подать заявку на кредит через АБС.                                |   Нужно учесть, что клиент мог уже получить предварительное одобрение по кредиту на сайте. Сотрудник должен продолжить работать с той же заявкой на кредит. Для полностью нового клиента он создает новую заявку.           |
| F9    | В интернет-банке клиент видит список доступных кредитов с актуальными условиями так же, как и на сайте                                |              |
| F10    | В интернет-банке клиенту может выводиться предодобренное предложение по кредиту                                |  то означает, что по клиенту в какой-то момент уже произошел скоринг и была рассчитана приемлемая сумма кредита            |
| F11    | В интернет-банке клиент, указав счёт зачисления средств и данные для подачи заявки, может отправить её в банк. Операция на подачу заявки на кредит подтверждается СМС-кодом                                |              |
|     | ...                                |              |
| U   | Удобство использования (Usability) |              |
| U1    | После подтверждения условий депозита менеджера бэк офиса в АБС, клиент должен получить СМС уведомение                                |              |
| U2    | Интерфейс сайта и интернет банка должен быть максимально удобным для пользователя, дизайн должен соответствовать системе дизайна компании                                |              |
|  U3   | Должно быть реализовано информирование клиента  по СМС об изменении статуса заявки на кредит                                 |              |
|     | ...                                |              |
| R   | Надёжность (Reliability)           |              |
| R1    | Режим работы 24/7                                |              |
| R2    | Сервисы должны быть доступны в 99,9% случаев                                |              |
|     | ...                                |              |
| P   | Производительность (Performance)   |              |
| P1    | Отклик по всем операциям должен быть максимально быстрым и занимать миллисекунды                                |              |
| P2    | Возможность горизонтального масштабирования                                |     АБС может масштабироваться только вертикально из-за своей базы данных         |
| P3    | Распределение  запросов между серверами, приложениями и ЦОД                                |              |
|     | ...                                |              |
|     | ...                                |              |
| S   | Поддерживаемость (Supportability)  |              |
|     | ...                                |              |
|     | ...                                |              |
| +R  | + Ограничения (Restrictions)       |              |
| +R1    | Требуется доработать функционал работы с СМС силами команды разработки банка                                |              |
| +R2    | Интернет-банк реализован на веб-фреймворке ASP.NET MVC 4.5 на основе .NET Framework 4.5 и СУБД MS SQL                               |              |
| +R3    | АБС. Интерфейс пользователей - десктопный клиент на Delphi и СУБД на Oracle. Основная логика работы системы реализована процедурами на PL-SQL в СУБД.                                |              |
| +R4    | Система кол-центра. Веб-интерфейс на React.js, бэкенд на Java Spring Boot и базы PostgreSQL                                |              |
| +R5    | Сайт. Бэкенд на PHP, фронтенд на ReactJS                                |              |
| +R6    | Кредитный конвеер. Коробочное решение на базе BPM-системы Camunda. База данных Oracle. Обаботчики бизнес процессов реализованы на Java                                |              |
| +R7    | При внедрении новых технологий необходимо учесть совместимость с существующими платформами разработки                                |              |
| +R8    | Перевести интернет-банк на микросервисную архитектуру в части задачи открытия депозитов. В качестве брокера сообщений использовать Kafka                                |              |
